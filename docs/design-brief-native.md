# BP Tracker — нативний застосунок: інвентар екранів і дизайн-бриф

Натив = «сьогодні» + скан + ліки/нагадування. Повна аналітика й редагування схем — у вебі.
Натив переймає **візуальну мову** веба (темна тема, неонові акценти зон, картковість, ті самі коди кольорів зон), але **менший набір екранів**.

**Маркування:** 🟢 робимо в дизайні й коді зараз · 🟡 дизайнимо зараз, кодимо пізніше · ⚪ лишається у вебі, в натив не тягнемо.

---

## A. Каркас / навігація
- 🟢 Нижній таб-бар (Дашборд / Розклад) + центральна кнопка **Скан**
- 🟢 Топ-бар: логотип + кнопка налаштувань (шестерня)
- 🟢 Глобальні стани: завантаження, помилка, порожньо
- 🟢 Тости/снекбари (підтвердження, помилки)
- 🟢 Діалог підтвердження (видалення тощо)
- 🟡 Офлайн-банер

## B. Дашборд «Сьогодні»
- 🟢 Картка останнього заміру (SYS/DIA/пульс, час, badge зони) — *є*
- 🟢 Незакриті прийоми ліків на сьогодні + швидкий «Прийняв»
- 🟡 Спарклайн-тренд у картці
- 🟡 KPI-плитки: середнє 7д, зміна за тиждень, % у нормі, середній пульс
- 🟡 Прев'ю останніх 2–3 замірів з лінком в історію
- 🟡 Мотиваційні лічильники: стрік днів у нормі, серія днів без пропуску ліків

## C. Вимірювання
- 🟢 **Ручне введення** заміру (форма + валідація)
- 🟢 **Скан тонометра** камерою + локальний OCR (ONNX)
- 🟢 Екран підтвердження OCR (редагувати SYS/DIA/пульс перед збереженням)
- 🟢 **Деталь одного заміру** + видалення
- 🟡 Прев'ю історії (останні N, групування по днях)
- 🟡 Офлайн-черга замірів (Room)
- ⚪ Повна історія з фільтром по періоду
- 🟢 **Експорт CSV на email** (кнопка → наявний `/export/csv`)  *(живе радше в Налаштуваннях/Історії)*

## D. Графіки / аналітика
- ⚪ Великий графік динаміки (СИС/ДІА/пульс, перемикачі 7д/30д/3м/рік)
- ⚪ Легенда, лінії норм

## E. Ліки / схеми / нагадування
- 🟢 **Розклад прийомів на сьогодні** + підтвердження
- 🟢 **Локальні нагадування** (AlarmManager) + «Прийняв» у нотифікації  *(механіка ще уточнюється — дизайн нотифікації пізніше)*
- 🟢 Перегляд активної схеми лікування (read-only)
- 🟡 Журнал прийомів / звіт виконання (стрік, % дисципліни)
- ⚪ Введення/редагування схем

## F. Дані / інтеграції
- 🟡 (?) Health Connect — *пізня фаза, дизайн пізніше*
- 🟡 (?) BLE — імпорт з Bluetooth-тонометра — *пізня фаза, дизайн пізніше*

## G. Налаштування
- 🟢 Тема (авто / світла / темна)
- 🟢 Мова (укр / англ)
- 🟢 Сервер / токен / вихід (logout)
- 🟢 Перемикач «надсилати фото для покращення моделі»
- 🟢 **Довідка: шкала тиску ESC** — таблиця рівнів + дисклеймер + **шкала «Норма → Небезпека» з позицією заміру**
- 🟡 Про застосунок / версія / оновлення APK

## H. Авторизація / вхід
- 🟢 Екран логіну (passkey) — *є*
- ⛔ Реєстрація паскея в нативі — *відкладено (окрема фаза)*
- 🟡 Онбординг (1–2 екрани) — *поліровка, у самому кінці*

---

## Що дизайнимо «на 100%» зараз (домен ясний)
Усі 🟢 + 🟡, **окрім** тих, де механіка ще не зафіксована: нотифікації нагадувань (E), Health Connect / BLE (F), онбординг (H). Їх — ескізом пізніше, коли визначиться поведінка.

## Дизайн-система (фундамент, спільний для всіх екранів)
- Токени: темна палітра + кольори зон (optimal/normal/stage1/stage2 — ті самі відтінки, що веб), типографіка, радіуси, відступи, тіні.
- Базові компоненти: `BpCard`, `ZoneBadge`, KPI-плитка, таб-бар з центральною кнопкою, поля форми, стани (loading/empty/error).
- Світла/темна теми (натив підтримує обидві, як веб).
